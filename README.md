Project Summary<br>
Customer churn represents a persistent challenge for organizations that rely on subscription based revenue models. When customers cancel services unexpectedly, organizations experience revenue loss and reduced customer lifetime value. Existing approaches to managing churn are often reactive, relying on historical reporting rather than predictive insight. This limits an organization’s ability to proactively identify and retain at-risk customers.<br>
The client for this project will be a subscription-based service organization seeking to improve customer retention through data-driven decision making. Middle management stakeholders will require a solution that provides insight without requiring deep technical expertise. The client’s primary need will be a practical tool that can identify customers who are likely to churn, enabling targeted retention strategies and more efficient allocation of resources.<br>
The proposed deliverables will include a completed machine learning application capable of predicting customer churn, a set of generated visualizations supporting descriptive and evaluative analysis, and a user guide describing how to execute and use the application. The application will be implemented as a stand-alone software solution and will produce both predictive outputs and supporting performance metrics.<br>
The proposed application will benefit the client by enabling proactive identification of churn risk, improving retention planning, and reducing revenue loss. By leveraging machine learning, the organization will gain insight into customer behavior patterns that are difficult to identify through manual analysis alone. The solution will emphasize interpretability and maintainability to support long term organizational adoption.<br><br>

Solution Summary<br>
The organizational problem addressed by this project was customer churn within subscription-based service organizations. Customer churn occurs when an existing customer discontinues service, resulting in lost revenue and missed opportunities for retention. Organizations often struggle to identify at-risk customers before churn occurs, which limits the effectiveness of retention strategies.<br>
To address this problem, a machine learning–based customer churn prediction application was developed. The application analyzed historical customer data and generated predictions indicating whether a customer was likely to churn. By providing churn risk predictions and probabilities, the application enabled proactive identification of at-risk customers and supported informed decision-making related to customer retention.<br>
The completed application directly addressed the organizational problem defined in Parts A and B by transforming raw customer data into actionable insights. Descriptive analytics were used to explore patterns in customer behavior, while predictive machine learning models were applied to forecast churn outcomes. The final application provided both performance metrics and user-driven predictions, demonstrating a practical and data-driven solution to the churn problem.<br><br>
Data Summary<br>
The raw data used in this project was sourced from a publicly available customer churn dataset obtained from an open data repository (Kaggle). The dataset represented anonymized customer records from a subscription-based telecommunications context and included demographic attributes, service usage details, billing information, and a labeled churn outcome. The data was collected and published for educational and analytical purposes and did not contain personally identifiable or restricted information.<br>
Throughout the application development lifecycle, the data was processed and managed in a structured and reproducible manner. During the design phase, the dataset was examined to assess data quality, feature availability, and suitability for supervised learning. In the development phase, data preprocessing steps were implemented to handle missing values, convert data types, encode categorical variables, and normalize numerical features. These steps were incorporated into a reusable preprocessing pipeline to ensure consistency during training, evaluation, and prediction.<br>
The dataset was stored locally in a flat file format and accessed programmatically by the application. No database management system was required due to the scope of the project. The preprocessing and modeling pipelines ensured that data handling remained consistent and maintainable across future executions of the application.<br><br>
Machine Learning<br>
Two supervised machine learning models were implemented in this project: Logistic Regression and a Decision Tree classifier.<br>
The Logistic Regression model was used as a baseline classification method. This model estimated the probability that a customer would churn based on weighted combinations of input features. The output of the model included a predicted churn class and an associated churn probability. For example, the model could indicate that a customer had a 0.65 probability of churn, signaling a higher retention risk.<br>
The Decision Tree model was implemented as a comparison method. This model learned a series of decision rules that segmented customers based on feature thresholds, allowing it to capture nonlinear relationships between customer attributes and churn outcomes. The model produced class predictions by traversing the learned decision structure.<br>
Both models were developed using a standardized machine learning pipeline. The data was split into training and testing subsets to ensure unbiased evaluation. Preprocessing steps, including feature scaling and categorical encoding, were applied consistently using a column transformer. Each model was trained on the same training data and evaluated on the same testing data.<br>
The selection of Logistic Regression was justified by its interpretability and effectiveness for binary classification problems. It provided a clear baseline and produced easily explainable outputs. The Decision Tree model was selected to demonstrate an alternative approach capable of modeling more complex feature interactions. Together, these models allowed for a meaningful comparison between interpretability and predictive flexibility.<br><br>
Validation <br>
Both machine learning models fell under the category of supervised learning, as they were trained using labeled historical data with known churn outcomes. Supervised learning was the most appropriate category for this application because the goal was to predict a specific outcome based on prior examples.<br>
An appropriate validation method was implemented using a train-test split strategy. The dataset was divided into training and testing subsets, with stratification applied to preserve class balance. Model performance was evaluated exclusively on the held-out test data to avoid overfitting and ensure generalization.<br>
For supervised learning, several classification performance metrics were used. These included accuracy, precision, recall, and F1-score. Accuracy measured overall correctness, precision measured the reliability of churn predictions, recall measured the model’s ability to identify actual churn cases, and F1-score provided a balanced measure of precision and recall.<br>
Testing results showed that the Logistic Regression model achieved an accuracy of approximately 80 percent, with balanced precision and recall values suitable for a churn prediction context. The Decision Tree model demonstrated comparable performance, with differences in precision and recall reflecting trade offs between model complexity and interpretability. Confusion matrix outputs were generated to visually represent classification performance and error types.<br><br>
